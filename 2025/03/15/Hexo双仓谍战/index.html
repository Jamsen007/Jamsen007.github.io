Q2：hexo deploy和git push有什么区别？

A：这是双线作战的秘密通道：

  • hexo deploy：把public/推送到公有仓（展示橱窗）
  • git push：把源码推送到私有仓（触发机器人自动工作）
    当使用GitHub Actions后，hexo deploy可退休了——机器人会自动帮你摆橱窗！

Q3：机器人怎么打通两个仓库？

A：通过以下黑魔法实现跨仓操控：

  1. 监听私有仓的git push事件
  2. 用HEXO_DEPLOY_KEY获得公有仓的写入权限
  3. 生成新的public/并强制推送到公有仓
    整个过程就像用私有仓的钥匙，打开了公有仓的保险柜！
步骤 私有仓操作 公有仓效果
1 推送源码到hexo-source 触发GitHub Actions
2 机器人读取Token 获得访问公有仓的权限
3 生成public/ 强制推送到yourname.github.io的main分支

Q4：为什么git push后GitHub Pages会自动更新？

A：因为GitHub Actions监听了你的推送事件，触发自动化部署流水线，就像有个机器人在帮你按部署按钮！
peaceiris/actions-gh-pages这个第三方插件会通过Token自动识别你的账户，把public/推送到<用户名>.github.io仓库

Q6：部署密钥到底放在哪？

A：HEXO_DEPLOY_KEY永远放在私有仓的Secrets里！

Q7：HEXO_DEPLOY_KEY 有什么作用？

A：HEXO_DEPLOY_KEY的真面目

  • 身份：这是你在GitHub生成的Classic Token
  • 超能力：拥有repo权限的Token可以操控你账户下的所有仓库
  • 存放位置：藏在私有仓的Secrets里（就像把核弹按钮放在自家保险柜）
