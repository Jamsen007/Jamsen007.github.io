Hexo实战指南：核心命令与高频问题解决方案

Hexo核心操作手册

一、生存必备命令集

基础操作三件套

命令 作用描述 典型场景
hexo init <folder> 初始化博客项目 开新坑时敲的第一行代码
hexo clean 清除缓存和生成文件 页面异常时优先执行
hexo generate 生成静态文件（可简写g） 本地调试前必须编译

内容管理三板斧

命令 作用描述 参数说明
hexo new "文章标题" 创建新文章 默认生成在_posts目录
hexo new draft "草稿标题" 创建草稿文件 存于_drafts的隐形模式
hexo publish "草稿标题" 发布草稿 让隐秘文件重见天日

调试部署二重奏

命令 作用描述 隐藏功能
hexo server 启动本地服务器 加--draft预览草稿
hexo deploy 部署到远程仓库 配合Git自动同步

高级玩家扩展包

命令 作用描述 使用场景
hexo list <type> 列出所有内容文件 查找失踪文章
hexo version 查看Hexo版本信息 排查版本兼容问题
npm install --force 强制重装node_modules 主题配置抽风时的终极手段

二、草稿系统实战手册

草稿工作流

  1. 创建隐身文档
  2. 本地预览
  3. 正式发布

注意事项

  • 草稿文件不会参与最终构建
  • 可配合_drafts目录进行版本管理
  • 建议定期清理3个月以上的陈年草稿

三、高频问题急救指南

问题1：部署后空白页面

典型症状
hexo d执行成功，但线上页面404

排查步骤

  1. 检查仓库分支是否设置为GitHub Pages服务分支
  2. 确认_config.yml的URL配置
  3. 查看部署目录是否存在

问题2：主题配置不生效

故障表现
修改主题设置后，本地预览无变化

解决方案

  1. 清除缓存
  2. 检查主题配置文件命名
  3. 查看主题版本兼容性

问题3：访问统计异常

常见现象
Busuanzi/LenaCloud计数不更新

诊断流程

  1. 检查页面元素
  2. 排查广告拦截插件
  3. LeanCloud控制台查看API调用日志
